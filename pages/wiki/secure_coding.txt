====== 시큐어 코딩 ======

===== 문서 개요 =====

  * 시큐어 코딩(secure coding)에 대한 개념을 정리한 문서
  * 시큐어 코딩를 유지하기 위해 지켜야 할 원칙들을 정리한 문서
 
===== 시큐어 코딩이란!? =====

  * 소프트웨어의 구현/개발 단계에서 해킹이나 적절하지 않은 입력 등의 취약점을 제거하여 높은 보안 수준의 소프트웨어를 개발하려는 개발 기법
  * "호미로 막을 수 있는 것을 미루다가 가래로 막는 일을 방지하자"는 문장으로 표현이 가능
  * 행정안전부가 정의한 취약점의 종류((호미로 막을 수 있는것))
    [[http://blog.lgcns.com/1152|{{ :wiki:secure_coding:hangan_vulnerability.jpg?nolink&700 |}}]]

===== 시큐어 코딩의 이점 =====

  * Java, C, 안드로이드 등등 개발 언어에 상관없이 추후에 크게 손봐야 할지도 모르는 보안 이슈를 초기 구현 단계에서 처리하여 비용 효율성을 제고할 수 있음

===== 시큐어 코딩, 어떻게 하나 =====

  * CERT((computer emergnecy response team, 컴퓨터 비상 대응팀))에서 제공하는 가이드가 있음[[https://www.securecoding.cert.org/confluence/display/seccode/CERT+Coding+Standards|가이드 링크]]
    * Android, C, C++, Java, perl 언어에 대한 가이드를 제공
    * 언어별로 시큐어 코딩을 하기위해 지켜야할 규칙들을 나열하고 각 규칙들의 내용은 예제를 들어서 설명
    * 몇몇 언어는 시큐어 코딩 규칙을 만족하기위해 추천하는 코딩 기법을 알려주고 있음
